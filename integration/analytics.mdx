## Overview

Mintie Calendar's Analytics API provides insights into your usage patterns, helping you understand task completion rates, event scheduling trends, and API consumption. You can also retrieve account information including your subscription plan and API quota.

## Get Analytics

Retrieve usage statistics for various time periods:

```bash
curl "http://localhost:8000/public/v1/analytics?period=last_30_days" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Available Periods

<ParamField query="period" type="string" default="last_7_days">
  Time period for analytics data

  **Options**:
  - `last_7_days` - Past week
  - `last_30_days` - Past month
  - `last_90_days` - Past quarter
  - `current_month` - Month to date
</ParamField>

## Analytics Response

```json
{
  "period": "last_30_days",
  "tasks_created": 127,
  "tasks_completed": 98,
  "tasks_active": 29,
  "events_created": 45,
  "upcoming_events": 12,
  "api_calls": 3450,
  "top_categories": ["Work", "Personal", "Meetings"]
}
```

## Usage Statistics

Get detailed usage metrics:

```bash
curl "http://localhost:8000/public/v1/analytics/usage" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Metrics Explained

<AccordionGroup>
  <Accordion title="Task Metrics">
    - **tasks_created**: Total tasks created in period
    - **tasks_completed**: Tasks marked as complete
    - **tasks_active**: Currently incomplete tasks
    - **Completion Rate**: `(tasks_completed / tasks_created) √ó 100%`
  </Accordion>

  <Accordion title="Event Metrics">
    - **events_created**: Calendar events scheduled
    - **upcoming_events**: Future events not yet passed
  </Accordion>

  <Accordion title="API Metrics">
    - **api_calls**: Total API requests made
    - **Average daily calls**: `api_calls / days_in_period`
  </Accordion>
</AccordionGroup>

## Account Information

Retrieve your account details, subscription plan, and API quota:

```bash
curl "http://localhost:8000/v1/account" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Response

```json
{
  "account_id": "demo_account",
  "email": "demo@example.com",
  "plan": "Professional",
  "created_at": "2025-08-15T10:00:00Z",
  "api_quota": 100000,
  "api_usage": 1523
}
```

### Account Fields

<ResponseField name="account_id" type="string">
  Unique identifier for your account
</ResponseField>

<ResponseField name="email" type="string">
  Account email address
</ResponseField>

<ResponseField name="plan" type="string">
  Current subscription plan (Free, Professional, or Enterprise)
</ResponseField>

<ResponseField name="created_at" type="datetime">
  When the account was created
</ResponseField>

<ResponseField name="api_quota" type="integer">
  Monthly API call limit based on your plan
</ResponseField>

<ResponseField name="api_usage" type="integer">
  API calls used in the current billing period
</ResponseField>

### Monitoring API Quota

Use this endpoint to track your API usage and avoid hitting rate limits:

```python
import requests

response = requests.get(
    "http://localhost:8000/v1/account",
    headers={"Authorization": f"Bearer {API_KEY}"}
)

account = response.json()
usage_percent = (account["api_usage"] / account["api_quota"]) * 100

print(f"API Usage: {account['api_usage']:,} / {account['api_quota']:,}")
print(f"Usage: {usage_percent:.1f}%")

if usage_percent > 80:
    print("‚ö†Ô∏è  Warning: Approaching API quota limit")
```
